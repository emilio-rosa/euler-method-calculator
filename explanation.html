How the Calculation Works

A step-by-step explanation of the mathematics behind Euler's method and exactly how the code turns that maths into numbers.

1 The Core Idea

A first-order differential equation like dy/dx = f(x, y) tells you the slope of the unknown curve y(x) at every point — but it does not tell you the curve itself directly.

Euler's method builds an approximation by starting at a known point (x₀, y₀) and repeatedly walking a small distance h along the current slope. Each step produces a new approximate point on the curve.

Analogy: Imagine you are hiking in fog and can only see the ground directly beneath your feet. You check your compass (the slope), take one small step in that direction, then check again. The smaller each step, the closer you stay to the true path.

2 Where the Formula Comes From

The definition of a derivative says that for small h:

( dy dx ) 0  ≈  y1 − y0 h
The slope at a point ≈ rise over run for a small step h

Rearranging to solve for y₁ gives us the Euler update rule:

yr+1  ≈  yr  +  h ( dy dx ) r    ,  r = 0, 1, 2, …
New y = old y + (step size) × (slope at old point)

At the same time x advances by one step: xr+1 = xr + h. These two updates together are the entire algorithm.

3 How the Code Does It

Below is a walkthrough of the key JavaScript functions in the calculator, matched with the maths they implement.

// Turn "2xy" into "2*x*y" function preprocessExpression(expr) { expr = expr.replace(/(\d)([a-zA-Z(])/g, '$1*$2'); expr = expr.replace(/([a-zA-Z])([a-zA-Z])/g, '$1*$2'); expr = expr.replace(/\^/g, '**'); return expr; }
Translate maths notation into JavaScript The user types natural maths like 2xy or x^2. JavaScript doesn't understand implicit multiplication or the caret, so this function rewrites them as 2*x*y and x**2 before the expression is evaluated.
// Compute f(x, y) at one point function evaluateSlope(expression, xValue, yValue) { const x = xValue; const y = yValue; const clean = preprocessExpression(expression); return new Function('x', 'y', 'return (' + clean + ')')(x, y); }
Evaluate the slope f(xᵣ, yᵣ) This is the right-hand side of the differential equation — the slope of the curve at the current point. new Function safely compiles and runs the user's expression as live JavaScript with x and y bound to the current values.
// The main Euler loop for (let r = 1; r <= totalSteps; r++) { const slope = evaluateSlope(expr, currentX, currentY); currentY = currentY + stepSize * slope; currentX = currentX + stepSize; points.push({ step: r, x: currentX, y: currentY }); }
The iterative update — the heart of Euler's method Each iteration applies the two update rules:

currentY += h · f(xᵣ, yᵣ)  →  yr+1 ≈ yr + h·(dy/dx)r
currentX += h  →  xr+1 = xr + h

The result of every step is saved so the table and graph can show the full path.
// Reference curve — built once, fixed const xRange = totalSteps * stepSize * 4; const fineH = xRange / 2000; const refPoints = runEulersMethod( expr, startX, startY, fineH, 2000 );
The dark purple "reference" curve This isn't the true analytical solution — it's Euler's method run once with 2000 very fine steps, covering 4× the initial x-range. It is computed only when the equation or initial conditions change, so it stays completely fixed as you drag the step-size slider. This lets you watch the blue Euler approximation improve or deteriorate relative to a stable reference line.

4 Accuracy and Step Size

Euler's method makes an error at every step because it walks in a straight line along the slope, while the true curve is bent. The error per step is roughly proportional to h², and since there are 1/h steps, the total accumulated error is proportional to h.

This means halving h roughly halves the final error — a first-order method. Use the slider in the calculator to see this directly:

h = 1.0

Very coarse — large visible gap between the two curves

h = 0.1

Reasonable — curves close but still a noticeable gap

h = 0.01

Fine — curves nearly overlap on screen

Why not always use a tiny h? More steps means more floating-point arithmetic, and rounding errors can accumulate. For most practical problems a step size between 0.01 and 0.1 gives a good balance of accuracy and speed.

5 What the (dy/dx)ᵣ Column Shows

The fourth column in the results table shows (dy/dx)r = f(xr, yr) — the slope of the approximated curve at each step point.

This value is what multiplies h in the update rule. A large slope means the curve is changing quickly and a large step will overshoot more. Watching the slope column grow (for equations like x + y) helps explain why accuracy degrades at later steps — the curve is bending faster than the straight-line steps can keep up.

6 Limitations of Euler's Method

Euler's method is the simplest numerical ODE solver but it has well-known weaknesses:

  • First-order accuracy — halving h only halves the error. Methods like Runge-Kutta 4 achieve fourth-order accuracy (error ∝ h⁴).
  • Instability — for some equations (especially stiff ones) the approximation can blow up even with small h.
  • One-directional — this implementation only steps in the positive x direction. Stepping backwards requires a negative h.
  • First-order ODEs only — higher-order equations (e.g. y″ = f) must be rewritten as a system of first-order equations first.
